聊天室已结束异常处理演示

🎯 问题描述

当聊天室状态为"已结束"时，用户仍然可以尝试发送消息、获取消息或标记已读，这会导致异常情况。我们需要在前端和后端都添加相应的处理机制。

🔧 解决方案

  • 后端API检查聊天室状态，返回410状态码和room_ended标志
  • 前端检测到聊天室已结束时，显示友好提示
  • 禁用输入框和发送按钮，防止继续操作
  • 添加视觉反馈，明确告知用户聊天室状态
  • 保持消息历史可见，但禁止新操作

💻 后端API响应示例

聊天室已结束时的API响应：

{ "success": false, "error": "聊天室已结束，无法发送消息", "room_ended": true }

HTTP状态码：410 Gone

使用410状态码表示资源已不存在，比404更准确地描述聊天室已结束的状态。

🎨 前端处理逻辑

// 发送消息时的处理 .then(data => { if (data.success) { // 正常处理 } else if (data.room_ended) { // 聊天室已结束，显示特殊提示并禁用发送功能 showRoomEndedMessage(); disableChatInput(); } else { // 其他错误处理 } })

✅ 修复效果

  • 用户无法向已结束的聊天室发送消息
  • 清晰的视觉提示告知用户聊天室状态
  • 输入框和按钮被禁用，防止误操作
  • 保持消息历史可见，用户可以查看之前的对话
  • 优雅的错误处理，不会出现异常或崩溃
